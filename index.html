Я – Євгеній Кравченко, студент групи LDE. Цей сайт створено для демонстрації моїх виконаних робіт у рамках assessment е-portfolio. Ви можете знайти мої відповіді на завдання, приклади робіт та інформацію про обрану платформу.

Завдання 1: Аналіз технологічних трендів

Завдання: Проаналізуйте сучасні технологічні тренди в розробці програмного забезпечення.

Відповідь:
Останніми роками найбільше впливають на розробку такі тренди: хмарні технології (Cloud), гнучка розробка (Agile), DevOps, розповсюдження мобільних платформ, автоматизація CI/CD та увага до безпеки. Я визначив для себе важливість вивчення роботи з Docker та Kubernetes, оскільки сучасні компанії активно використовують контейнеризацію. Також слідкував за трендами в AI, але для даного assessment AI не використовував.

Завдання 3: Реальний кейс з розробки

Завдання: Опишіть кейс, де ви застосовували сучасний стек технологій.

Відповідь:
Під час навчального проєкту я створював веб-сайт з використанням HTML, CSS, JavaScript та фреймворку React. Я застосовував Git для контролю версій, працював у команді через GitHub, що дозволило швидко виявити та виправити помилки. Для деплойменту обрали Netlify, а для дизайну — Figma. Такий підхід допоміг виконати завдання у визначений термін й чітко розподілити ролі.

Завдання 4: Саморефлексія та оцінка процесу

Завдання: Самоаналіз: що вдалося, а що варто покращити?

Відповідь:
Я навчився краще структурувати власну роботу, чітко планувати виконання, шукати інформацію і взаємодіяти із онлайн-інструментами. Вдалося підвищити навички у пошуку рішень для нетипових задач. Ще треба покращити тайм-менеджмент на етапі детального планування завдання. Також планую довчити більш глибоко основи безпеки вебу.

Ілюстративний огляд інструменту

Використана платформа: Я створював портфоліо як статичний сайт (HTML, CSS, JS), щоб мати максимальний контроль, незалежність та відповідність усім вимогам assessment.

  • Створив структуру сайту, згідно з умовами (головна, 4 завдання, огляд)
  • Інтегрував всі відповіді, ілюстрації, скріншоти та лінки
  • Забезпечив легкий доступ (скріншоти, посилання відкриті для викладача)
  • Перевірив роботу у приватному режимі
